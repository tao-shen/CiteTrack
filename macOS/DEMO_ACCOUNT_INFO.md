# CiteTrack - App Store 审核演示账户信息

## 应用功能概述

CiteTrack 是一个 macOS 应用，用于跟踪 Google Scholar 上的学术引用数据。应用的主要功能包括：

1. **添加学者跟踪**：用户可以输入 Google Scholar ID 来跟踪特定学者的引用数据
2. **引用数据展示**：显示学者的总引用数、H-index 等指标
3. **图表可视化**：提供引用历史趋势图表
4. **iCloud 同步**：支持通过 iCloud 在多设备间同步数据
5. **通知提醒**：当引用数据更新时发送通知

## 演示账户说明

**重要提示**：CiteTrack 不需要用户登录或创建账户。应用使用 Google Scholar 的公开数据，只需要 Google Scholar ID（公开的学者标识符）即可使用。

### 测试步骤

1. **启动应用**
   - 应用启动后会显示主窗口
   - 如果首次使用，学者列表为空

2. **添加学者跟踪**
   - 点击"添加学者"或"+"按钮
   - 输入一个有效的 Google Scholar ID（例如：`kukA0LcAAAAJ`）
   - 或者输入完整的 Google Scholar 个人资料 URL
   - 点击"添加"按钮

3. **查看引用数据**
   - 添加后，应用会自动获取该学者的引用数据
   - 在主界面可以看到：
     - 学者姓名
     - 总引用数
     - 最后更新时间
   - 点击学者条目可以查看详细信息

4. **查看图表**
   - 点击"查看图表"或菜单中的"图表"选项
   - 可以查看引用历史趋势图
   - 支持多种图表类型（折线图、柱状图等）

5. **设置功能**
   - 打开"设置"窗口
   - 可以配置：
     - 更新间隔
     - 是否显示在 Dock
     - 是否显示在菜单栏
     - 是否开机启动
     - iCloud 同步设置

6. **iCloud 同步**（可选）
   - 在设置中启用 iCloud 同步
   - 需要用户登录 iCloud 账户
   - 数据会自动同步到 iCloud Drive

### 推荐的测试 Google Scholar ID

以下是一些公开的 Google Scholar ID，可用于测试：

- `kukA0LcAAAAJ` - 示例学者 ID
- 任何公开的 Google Scholar 个人资料 ID

### 注意事项

1. **网络要求**：应用需要网络连接来获取 Google Scholar 数据
2. **数据来源**：所有数据来自 Google Scholar 公开页面，不需要 API 密钥
3. **隐私**：应用不收集用户个人信息，只跟踪用户指定的公开学者数据
4. **iCloud 同步**：如果测试 iCloud 功能，需要审核设备登录 iCloud 账户
