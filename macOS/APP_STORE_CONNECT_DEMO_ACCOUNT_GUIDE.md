# App Store Connect 演示账户填写指南

## 重要提示

CiteTrack 应用**不需要用户登录或创建账户**。应用使用 Google Scholar 的公开数据，只需要 Google Scholar ID（公开的学者标识符）即可使用。

## 在 App Store Connect 中填写演示账户信息

### 步骤 1：登录 App Store Connect

1. 访问 [App Store Connect](https://appstoreconnect.apple.com/)
2. 使用你的开发者账户登录

### 步骤 2：进入应用页面

1. 点击 **"我的 App"** (My Apps)
2. 选择 **CiteTrack** 应用

### 步骤 3：进入 App 审核信息

1. 在左侧菜单中，找到 **"App 信息"** (App Information)
2. 向下滚动到 **"App 审核信息"** (App Review Information) 部分
3. 找到 **"演示账户"** (Demo Account) 字段

### 步骤 4：填写演示账户信息

在 **"演示账户"** (Demo Account) 文本框中，**直接复制粘贴以下内容**：

```
CiteTrack 不需要用户登录或创建账户。应用使用 Google Scholar 的公开数据，只需要 Google Scholar ID（公开的学者标识符）即可使用。

测试步骤：
1. 启动应用
2. 点击"添加学者"或"+"按钮
3. 输入以下任一 Google Scholar ID：
   - kukA0LcAAAAJ
   - 或任何其他公开的 Google Scholar 个人资料 ID
4. 点击"添加"按钮
5. 应用会自动获取并显示该学者的引用数据

应用功能：
- 查看学者的总引用数、H-index 等指标
- 查看引用历史趋势图表
- 导出数据（JSON/CSV 格式）
- iCloud 同步（如果启用）
- 通知提醒

所有功能都可以通过输入 Google Scholar ID 来测试，无需任何账户或登录。
```

### 步骤 5：保存并提交

1. 点击 **"保存"** (Save) 按钮
2. 确保信息已保存成功

## 如果字段是必填的

如果 App Store Connect 要求必须填写用户名和密码，可以这样填写：

- **用户名/邮箱**：`demo@citetrack.app` 或 `无需登录`
- **密码**：`无需密码`
- **说明**：在备注或说明字段中，填写上面的详细说明

## 验证

填写完成后，在重新提交应用时，Apple 审核人员应该能够看到这些信息。

## 注意事项

- 确保信息清晰、完整
- 使用简单明了的语言
- 提供具体的测试步骤
- 说明应用不需要登录即可使用
